# gRPC Service Documentation

## Service: LocationService

This service provides location data for a given person within a date range.

### Endpoint

GetLocations(LocationRequest) returns (LocationResponse)

### Request Message

* person_id (int): ID of the person
* start_date (string): Start date (YYYY-MM-DD)
* end_date (string): End date (YYYY-MM-DD)

### Response Message

* locations (list): List of Location objects containing:

  * person_id
  * longitude
  * latitude
  * creation_time

---

## Example Python Client

```python
import grpc
import location_pb2
import location_pb2_grpc

channel = grpc.insecure_channel('location-service:50051')
stub = location_pb2_grpc.LocationServiceStub(channel)

request = location_pb2.LocationRequest(
    person_id=1,
    start_date="2020-01-01",
    end_date="2020-12-31"
)

response = stub.GetLocations(request)

for loc in response.locations:
    print(loc)
```

---

## Notes

* The gRPC server runs on port 50051 inside the Kubernetes cluster.
* The service name `location-service` is resolved via Kubernetes DNS.
* This service replaces direct database access from the API service.
