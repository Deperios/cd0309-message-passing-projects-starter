==============================
ARCHITECTURE DECISIONS
==============================

1. REST (API Service)
------------------------------
REST is used as the external interface because it is simple, widely supported, 
and fast to implement. This aligns with the requirement to deliver a Minimum 
Viable Product (MVP) within a short timeframe and with a limited budget.


2. gRPC (Location Service)
------------------------------
gRPC is used for internal communication between services because it provides 
high performance and low latency. This is important for efficiently processing 
location proximity calculations at scale.


3. Kafka (Ingestion + Data Service)
------------------------------
Kafka is used to handle high-volume data ingestion asynchronously, allowing 
the system to scale and process large amounts of location data without 
overwhelming the services.
